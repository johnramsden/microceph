The one regression

Caught independently by the quality army (H1) and the equivalence completeness critic — and missed by the RGW per-group comparator, which reported zero diffs. Confirmed by hand against 2ff778a.

Regression · blocking

Exercise RGW Head Node

migration added a raising step · resources/microceph_harness.resource:169

Exercise RGW Head Node
    Run In Container    node-wrk0    microceph.ceph status    30
+   Run In Container    node-wrk0    sudo systemctl status snap.microceph.rgw --no-pager    30
    ${user_exists}=    Run In Container Unchecked    node-wrk0    ... user list | grep -q test ...
    # old (2ff778a): no systemctl step — zero occurrences in this keyword
Why it breaks

Run In Container is the checking variant — it raises on non-zero rc. systemctl status exits 3 when the unit is inactive. The post-upgrade caller Test Exercise RGW After Upgrade (upgrade_reef_tests.robot:53) runs this with no RGW-ready gate in front — the prior test only waits for OSDs and health. A momentarily-inactive RGW after a rolling upgrade now fails where the old keyword proceeded.

Fix

Either gate the post-upgrade call with Wait For RGW On Head Node, or change the inserted step to Run In Container Unchecked so it stays diagnostic and non-raising — matching the baseline's behavior. A deliberate daemon-health gate, if wanted, belongs in its own PR, not a migration.

New code · resource leak

copy_source_to_vm

git Popen not in try/finally; return code unchecked · microceph_harness.py:968-977

Why it matters

git archive is started with Popen; git.stdout.close() / git.wait() run only after a subprocess.run(..., timeout=120). A TimeoutExpired on a large repo skips both and orphans the git process. Separately, git.returncode is never checked — a partial archive can let tar succeed on truncated source and mislead later failures.

Fix

Wrap the run in try / finally that always closes the pipe and waits, then raise on a non-zero git.returncode. This is brand-new code — the old shell pipeline had no Popen to leak.

What the refactor got right

Design wins

01

Purify, done right on the new surface

The rbd helpers and the _network_cidr / _ceph_osd_counts / _rgw_daemon_count family fetch raw output and decide in pure, self-free, BuiltIn-free Python — exactly the spec's intent.

02

86 pytest cases with no LXD

The pure helpers now have real, fast regression coverage runnable from the unit-tests suite — something the monolithic resource never had.

03

Library rules honored

Lowercase class==module, ROBOT_LIBRARY_SCOPE = "SUITE", lazy variable reads, the load-bearing .rc result tuple, one shared _poll_until, and no BuiltIn().run_keyword.

04

Clean module separation

Area modules are imported at the Python level without re-exporting keyword names — avoiding Robot's duplicate-keyword errors.
